# Quick Deployment Steps

## 1. Push to GitHub

```bash
# Create a new repository on GitHub (https://github.com/new)
# Name it something like: estateai-backend

# Then run:
git remote add origin https://github.com/YOUR_USERNAME/estateai-backend.git
git branch -M main  
git push -u origin main
```

## 2. Deploy to Railway

1. Go to https://railway.app and sign in
2. Click "New Project"
3. Select "Deploy from GitHub repo"
4. Choose your `estateai-backend` repository
5. Railway will automatically detect it's a NestJS app

## 4. Wait for Deployment

Railway will:
- Install dependencies
- Build the app
- Run Prisma generate
- Start the server

Watch the logs in Railway dashboard.

## 5. Run Database Migrations

Once deployed, click on your service → Settings → "Run a command":

```bash
npx prisma migrate deploy
```

## 6. Create Admin User

In Railway, run:

```bash
node create-admin.js
```

This will create the default admin user.

## 7. Update WhatsApp Webhook

1. Go to Meta Developer Console
2. Update webhook URL to: `https://your-app.railway.app/api/webhooks/meta/whatsapp`
3. Verify token: `estate_verify_token_2025`

## 8. Update BACKEND_URL

After Railway generates your URL:
1. Copy the URL (e.g., `https://estateai-production.up.railway.app`)
2. Add it as environment variable: `BACKEND_URL=https://your-app.railway.app`
3. Redeploy

## 9. Test Everything

## Troubleshooting

If deployment fails:
1. Check Railway logs
2. Verify all environment variables are set
3. Ensure DATABASE_URL is correct
4. Check if migrations ran successfully

If WhatsApp doesn't work:
1. Verify webhook URL in Meta dashboard
2. Check META_WA_TOKEN is valid
3. Test webhook: Send a message and check Railway logs
4. Ensure BACKEND_URL is set correctly
