# Railway Deployment Guide

## Prerequisites
1. GitHub account with repository
2. Railway account (sign up at https://railway.app)
3. Database URL (Neon PostgreSQL recommended)

## Step 1: Push to GitHub

```bash
# Create a new repository on GitHub first, then:
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git
git branch -M main
git push -u origin main
```

## Step 2: Deploy to Railway

### Option A: Deploy via Railway Dashboard
1. Go to https://railway.app
2. Click "New Project"
3. Select "Deploy from GitHub repo"
4. Choose your repository
5. Railway will auto-detect the NestJS app

### Option B: Deploy via Railway CLI
```bash
# Install Railway CLI
npm install -g @railway/cli

# Login
railway login

# Initialize project
railway init

# Deploy
railway up
```

## Step 3: Configure Environment Variables

In Railway dashboard, add these environment variables:

### Database
```
DATABASE_URL=your_neon_postgresql_url
```

### JWT
```
JWT_SECRET=your-super-secret-jwt-key-change-this-in-production
JWT_EXPIRES_IN=24h
```

### WhatsApp (Meta Cloud API)
```
WHATSAPP_PROVIDER=meta
META_WA_PHONE_NUMBER_ID=your_phone_number_id
META_WA_TOKEN=your_access_token
META_WA_VERIFY_TOKEN=your_verify_token
```

### AI Configuration
```
GEMINI_API_KEY=your_gemini_api_key
```

### Image Hosting
```
IMGBB_API_KEY=your_imgbb_api_key
```

### URLs (will be auto-generated by Railway)
```
BACKEND_URL=https://your-app.railway.app
FRONTEND_URL=https://your-frontend-url.vercel.app
```

## Step 4: Run Database Migrations

After deployment, run migrations:

```bash
# Via Railway CLI
railway run npx prisma migrate deploy

# Or via Railway dashboard
# Go to your service → Settings → Add a one-off command:
cd backend && npx prisma migrate deploy
```

## Step 5: Create Admin User

```bash
# Via Railway CLI
railway run node backend/create-admin.js

# Or SSH into Railway container
railway shell
node backend/create-admin.js
```

## Step 6: Update WhatsApp Webhook

Update your Meta WhatsApp webhook URL to:
```
https://your-app.railway.app/api/webhooks/meta/whatsapp
```

Verify token: `estate_verify_token_2025` (or your custom token)

## Step 7: Test Deployment

1. Check health endpoint:
   ```
   https://your-app.railway.app/health
   ```

2. Test WhatsApp webhook:
   ```
   https://your-app.railway.app/api/webhooks/meta/whatsapp/health
   ```

3. Test API:
   ```
   https://your-app.railway.app/api/estates
   ```

## Troubleshooting

### Build Fails
- Check Railway logs in dashboard
- Ensure `package.json` has correct scripts
- Verify Node version compatibility

### Database Connection Issues
- Verify DATABASE_URL is correct
- Check if Neon database is accessible
- Run migrations: `railway run npx prisma migrate deploy`

### WhatsApp Not Working
- Verify webhook URL in Meta dashboard
- Check META_WA_TOKEN is valid
- Ensure BACKEND_URL is set correctly
- Check Railway logs for errors

### Image Upload Fails
- Verify IMGBB_API_KEY is set
- Test image upload: `railway run node backend/test-image-upload.js`
- Check network connectivity

## Monitoring

### View Logs
```bash
railway logs
```

### Check Metrics
- Go to Railway dashboard
- View CPU, Memory, Network usage
- Set up alerts for errors

## Scaling

Railway auto-scales based on usage. For manual scaling:
1. Go to Service Settings
2. Adjust resources
3. Save changes

## Cost Optimization

- Railway free tier: $5/month credit
- Estimated cost: $5-10/month for small estates
- Monitor usage in dashboard
- Optimize database queries
- Use caching where possible

## Backup Strategy

### Database Backups
- Neon provides automatic backups
- Export manually: `pg_dump DATABASE_URL > backup.sql`

### Code Backups
- GitHub repository (already done)
- Tag releases: `git tag v1.0.0 && git push --tags`
